# 🎮 Escape from Epstein Island: 2D Roguelike-платформер на SFML

Добро пожаловать в **Escape from Epstein Island** — игру в жанре roguelike с видом сверху, где вы исследуете процедурно генерируемые подземелья, сражаетесь с врагами и стараетесь выжить как можно дольше. Проект написан на C++17 с использованием библиотеки SFML и демонстрирует продвинутую модульную архитектуру (нет).

---

## 🧩 Особенности

- **Процедурная генерация** карты с помощью BSP-дерева (Binary Space Partitioning)
- **Случайное расположение** комнат, коридоров, стен и врагов
- **Гибкая система сущностей**: игрок, враги, пули, стены
- **Реалистичная стрельба** с перезарядкой и расчётом попаданий
- **Коллизии** на основе ограничивающих прямоугольников (AABB)
- **Динамическая загрузка ресурсов** через ResourceManager (можно было и через enum и массивы в compile-time, но я говнокодер, сори)
- **Возможность расширения** под многопользовательский режим (встроен HTTPS-сервер/клиент на базе **httplib**) с возможностью
2FA авторизации через TOTP.

## 🧠 Ключевые компоненты

### 🚀 Ядро (`Engine`)
Центральный класс, который управляет главным игровым циклом, обновлением всех сущностей, отрисовкой и обработкой событий. Инициализирует окно SFML, загружает текстуры и запускает игру.

### 🤺 Игрок (`Player`)
Наследуется от `Entity`. Имеет:
- перемещение с помощью клавиш (WASD/стрелки)
- систему стрельбы с задержкой (`ShootDelay`)
- уменьшенный хитбокс для более честных попаданий
- метод `cooldown()` для сброса таймера после выстрела

### 👾 Противники (`Enemy`)
Также наследуются от `Entity`:
- движутся по направлению к игроку
- имеют здоровье и получают урон
- проверяют столкновение с пулями игрока
- уничтожаются при нулевом здоровье

### 🔨 Генератор подземелий (`DungeonGenerator`)
Ключевой модуль процедурной генерации:
- Использует **BSP‑дерево** (`BSPNode`) для рекурсивного деления пространства
- Преобразует листовые узлы в комнаты (`Room`)
- Строит сетку (`RoomGrid`) и создаёт объекты `Room` с передачей чертежа (`blueprint`)
- Позволяет задавать глубину дерева, минимальный размер комнат и узлов
- Результат работы сохраняется в `DungeonData`

### 🏠 Комнаты (`Room`)
- Хранят внутреннее представление в виде двумерной сетки (`blueprint`)
- Содержат стены (`Wall`), которые также являются наследниками `Entity`
- Предоставляют шаблонный метод `checkCollision()` для проверки пересечений с любыми объектами, имеющими `getHitbox()`
- Участвуют в отрисовке и обновлении всех своих элементов

### 🧱 Стены (`Wall`)
- Неподвижные объекты, создаваемые внутри комнат
- Участвуют в коллизиях – не дают игроку и врагам проходить сквозь них

### 💥 Пули (`Bullet`)
- Автоматически летят в заданном направлении
- Ограничены временем жизни (`lifetime`)
- Наносят урон врагам при попадании
- Могут быть деактивированы (`setBulletAlive`)

### 📦 Менеджер ресурсов (`ResourceManager`)
- Кеширует загруженные текстуры для избежания повторной загрузки
- Единственная точка получения графических ресурсов

### ⏱️ Контекст обновления (`UpdateContext`)
- Удобный контейнер, передающий в методы `update()` время кадра, ссылку на окно и текущую комнату
- Упрощает сигнатуры обновления сущностей

## ⚙️ Зависимости

- **SFML 2.6** — для графики, ввода и окон.
- **C++17** — используется в проекте (автоматический вывод типов, `std::optional` и т.д.).
- **httplib** (опционально, встроена) — для создания встроенного сервера/HTTPS-клиента.

Установка SFML на Ubuntu/Debian:

```bash
sudo apt install libsfml-dev
```

На Windows проще всего использовать пакет через vcpkg.

## 🔧 Сборка и запуск

```bash
git clone <ваш-репозиторий>
cd <project-folder>
mkdir build && cd build
cmake ..
cmake --build .
./game
```

Хочу сразу предупредить: если вы хотите протестировать игру без регистрации, 
то измените значение макроса **DEBUG_NOT_AUTH** на 1
